# 项目文档

## 目标

本工具将装潢、空调、IC卡、监控和 LCD 采购记录的重复工作拆为可独立执行的步骤：Excel 整理、网页附件查询、附件回填、PM Tracking 匹配，以及价格计算。五条业务线共用浏览器查询核心，但各自使用独立的下载目录和待回填清单。

## 本地输入与输出

运行前按工作台提示准备采购记录、工厂映射、NI PM Tracking 和价格表。当前默认文件名为 `采购记录0701.xlsx`、`工厂清单 7.XLSX`、`NI PM Saving Tracking.xlsx`；空调、IC卡、监控、LCD 价格表均由按钮⑥选择。它们均属于业务数据，不进入 Git。

| 内容 | 位置/结果 |
| --- | --- |
| 登录状态 | `auth.json`，首次 SSO 登录后自动生成 |
| 已下载附件 | `downloads/<上月数据>/<品类>/FLD文件/<PO>/` 或 `无FLD文件/<PO>/` |
| 查询待回填清单 | `downloads/<上月数据>/<品类>/待回填任务.json` |
| 日志与报告 | 项目目录中的日志/报告文件，仅本地保留 |

FLD 的判断以 PO 文件夹内的实际附件为准：存在符合规则的 FLD 邮件附件即为 FLD；第五列的普通附件名称不会改变该判断。

## 工作台操作

| 操作 | 作用 | 可重复执行性 |
| --- | --- | --- |
| 匹配区域/分公司 | 将工厂信息写入源数据 | 数据更新后执行 |
| ① 生成透视表 | 生成当前品类的原生透视表 | 会重建对应透视表 |
| ② 添加扩展列 | 增加 E2E、价格、Saving 等业务列 | 已有列不会重复创建 |
| ③ 打开网站查询 | 仅查询 PO、下载附件并保存待回填清单，不写 Excel | 可按需要重跑 |
| ↻ 补查缺失 PO | 先核对预期 PO 与现有文件夹，只重新查询缺失项 | 不重跑已完成 PO |
| ④ 回填已下载文件 | 解析已下载附件并写入当前品类透视表 | 清单成功后归档，失败可修复后重试 |
| ⑤ 匹配 PM Tracking | 以 E2E 项目名匹配后先筛选 Content，再写入 Price、PlanCost、总 Saving、订单差异 | 可重复执行；多条歧义匹配留给人工 |
| ⑥ 填充空调/IC卡/监控/LCD老/新价格 | 空调、IC卡、监控或 LCD：填充老价格、新价格、Saving、CHECK；真正的 FLD PO 以 PM 结果为准并分组合并 | 无 FLD 的人工结果不会被重复覆盖 |

PM 匹配结束时，程序会比较透视表 `Price` 与 `项目总金额(审批)`，允许 0.01 的四舍五入误差；不一致项会写入报告，供人工处理。

## 运行与验证

```powershell
pip install -r requirements.txt
playwright install msedge
python 装潢透视表工具.py

py -3.8 -m unittest discover -s tests -v
py -3.8 -m py_compile 装潢透视表工具.py web_query.py
```

关闭 Excel 后再让程序写入；首次查询需在 Edge 中完成 SSO。查询和回填已拆开，网页阶段耗时较长时可先结束查询、稍后单独执行回填。

## 维护与回退

不再保留 `.bak` 脚本副本。每次完成一组可验证改动后创建 Git 提交；重要稳定版本再创建标签。

```powershell
git status
git log --oneline --decorate
git tag

# 只恢复一个文件到稳定标签，不影响其他未提交内容
git restore --source v1.0.0 -- web_query.py
```

发布前只提交代码、测试、文档和依赖清单；业务 Excel、附件、登录状态、日志和报告已由 `.gitignore` 排除。
