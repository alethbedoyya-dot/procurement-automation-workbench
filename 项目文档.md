# 项目文档

## 目标

本工具将九个采购品类的重复工作拆为可独立执行的步骤：Excel 整理、网页附件查询、附件回填、PM Tracking 匹配与价格计算。所有品类共用同一套稳定的浏览器查询核心，但各自使用独立下载目录与待回填清单，避免相互覆盖。

| 流程组 | 品类 | 主要结果 |
| --- | --- | --- |
| 透视表 + PM | 装潢、外包板 | 透视表、附件回填、PM Tracking |
| 透视表 + PM + 价格 | 空调、IC卡、监控、LCD | 在 PM 结果基础上完成老价格、新价格、Saving、CHECK |
| 轻量价格表 | 线槽、无线五方通话 | 独立数据 Sheet，填充老价格、新价格、Saving |
| 照明汇总 | 井道照明 | 独立数据 Sheet、描述/PO 数量与 Saving 汇总表 |

## 本地输入与输出

运行后，首次点击任意业务按钮时选择当月采购记录；文件必须包含 `Sheet1`，本次工作台运行会始终使用同一份已选文件。工厂映射、NI PM Tracking 与价格表按工作台提示准备；价格表由相应品类的按钮⑥选择。它们均属于业务数据，不进入 Git。

| 内容 | 位置/结果 |
| --- | --- |
| 登录状态 | `auth.json`，首次 SSO 登录后自动生成 |
| 已下载附件 | `downloads/<上月数据>/<品类>/FLD文件/<PO>/` 或 `无FLD文件/<PO>/` |
| 查询待回填清单 | `downloads/<上月数据>/<品类>/待回填任务.json` |
| 日志与报告 | 项目目录中的日志/报告文件，仅本地保留 |

FLD 的判断以 PO 文件夹内的实际附件为准：存在符合规则的 FLD 邮件附件即为 FLD；第五列的普通附件名称不会改变该判断。

## 工作台操作

| 操作 | 作用 | 可重复执行性 |
| --- | --- | --- |
| 匹配区域/分公司 | 将工厂信息写入源数据 | 数据更新后执行 |
| ① 生成数据/透视表 | 生成当前品类配置的数据 Sheet 或原生透视表 | 会重建当前品类的结果 Sheet |
| ② 添加扩展列 | 按当前流程增加 WBS、E2E、PM 或价格计算需要的业务列 | 已有列不会重复创建 |
| ③ 打开网站查询 | 查询 PO、读取详情页 WBS、下载附件并保存待回填清单，不写 Excel | 可按需要重跑 |
| ↻ 补查缺失 PO | 先核对预期 PO 与现有文件夹，只重新查询缺失项 | 不重跑已完成 PO |
| ④ 回填已下载文件 | 所有 PO 回填 WBS；FLD 附件再解析并写入相应数据/透视表 | 清单成功后归档，失败可修复后重试 |
| ⑤ 匹配 PM Tracking | 适用于装潢、外包板、空调、IC卡、监控、LCD；以 E2E 项目名匹配后再筛选 Content，写入 Price、PlanCost、总 Saving、订单差异 | 可重复执行；多条歧义匹配留给人工 |
| ⑥ 填充价格与 Saving | 空调、IC卡、监控、LCD 填充老价格、新价格、Saving、CHECK；线槽和无线五方通话仅填前三列 | 无 FLD 的人工结果不会被重复覆盖 |

PM 匹配结束时，程序会比较透视表 `Price` 与 `项目总金额(审批)`，允许 0.01 的四舍五入误差；不一致项会写入报告，供人工处理。

无线五方通话是唯一按物料号取价的品类：`五方通话数据.物料` 精确匹配价格表 `PN(Material)`，并读取 H 列老价格、J 列新价格。其余轻量品类仍按各自既定的短文本规则匹配，避免名称相近时误写价格。

## 运行与验证

日常使用时，直接双击 `启动采购自动化.bat`。它会优先使用 `py -3`，其次使用 `python`，并在首次运行时自动创建项目目录下的 `.venv`、安装/更新 `requirements.txt` 中的依赖以及 Edge 自动化组件。首次需联网，之后无需 VSCode 或终端；当依赖清单变化时，会自动重新安装依赖。

不要复制其他电脑的 `.venv` 或 `auth.json`。每台电脑都应通过启动文件创建自己的运行环境，并用自己的公司账号完成首次登录。

维护或验证代码时可使用：

```powershell
pip install -r requirements.txt
playwright install msedge
python 装潢透视表工具.py

py -3.8 -m unittest discover -s tests -v
py -3.8 -m py_compile 装潢透视表工具.py web_query.py
```

关闭 Excel 后再让程序写入；首次查询需在 Edge 中完成 SSO。查询和回填已拆开，网页阶段耗时较长时可先结束查询、稍后单独执行回填。

## 维护与回退

不再保留 `.bak` 脚本副本。每次完成一组可验证改动后创建 Git 提交；重要稳定版本再创建标签。

```powershell
git status
git log --oneline --decorate
git tag

# 只恢复一个文件到稳定标签，不影响其他未提交内容
git restore --source v1.0.0 -- web_query.py
```

发布前只提交代码、测试、文档和依赖清单；业务 Excel、附件、登录状态、日志和报告已由 `.gitignore` 排除。
