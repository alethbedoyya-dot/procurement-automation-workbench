```
装潢旧版/
├── 装潢透视表工具.py          ← 主程序（GUI）
├── web_query.py               ← 网站自动化 + PDF提取

```

## 按钮与流程

| 按钮 | 流程 |
|------|------|
| 🟢 **生成装潢透视表** | 读工厂清单 → Sheet1填区域/分公司 → 筛选物料 → COM生成原生透视表 |
| 🔵 **添加扩展列** | 透视表右侧追加9列空表头 |
| 🟣 **打开网站查询** | Edge登录 → 逐PO搜索 → 详情页提取 → 下载附件 → PDF解析 → 写回Excel |
| 🟢 **匹配PM Tracking** | E2E项目名模糊匹配 → 填Price/PlanCost → 计算总saving/差异 |

## 网站查询流程

```
提取PO号（测试模式前5个）
  → Edge登录（首次手动SSO，后续auth.json自动）
  → 循环每个PO：
      填PO → 查找 → 点请求ID → 详情页
      → 提取：项目名称 / 梯台明细合计×数量
      → 下载附件到 downloads/项目名/
  → 全部跑完：
      PDF解析(extract_text+tables, 自动摆正旋转页)
      → 搜「总计（不含税）」提取金额
      → 差异 = 总金额 - 订单净值（为0填0）
      → 写回Excel（按采购凭证行对齐）
```

## 关键机制

- **登录**：Microsoft SSO，auth.json持久化，过期自动提示重登
- **按钮定位**：10种选择器 + Enter键兜底
- **防休眠**：搜索页复用不关闭
- **PDF**：子进程45秒超时 + extract_tables补充表格文字 + 自动摆正旋转页
- **附件**：项目名分子文件夹，非法字符替换为`_`
- **弹窗**：成功/失败计数 + 列出未获取金额的项目

## 注意事项

1. 运行前关闭Excel中的 EXPORT1 2 1.xlsx
2. 工厂清单：A=Plant, J=区域, K=分公司
3. 首次网站查询需手动完成 Microsoft 登录（最长等5分钟）
4. 测试模式：改 `all_pos[:5]` 可调整PO数量
5. 回滚：复制 `web_query_backup.py` 覆盖 `web_query.py`
