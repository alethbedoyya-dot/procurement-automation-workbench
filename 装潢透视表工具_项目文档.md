## 网站查询流程

```
提取全部PO号（正式模式，不再截取前N个）
  → Edge登录（首次手动SSO，后续auth.json自动）
  → 循环每个PO：
      填PO → 查找 → 点请求ID → 详情页
      → 下载全部附件（不再仅限FLD）
        ├─ FLD 开头 → downloads/FLD文件/<PO号>/
        └─ 非 FLD   → downloads/无FLD文件/<PO号>/
      → 有 FLD 附件时：
          提取：项目名称 / 梯台明细合计×数量
      → 无 FLD 附件时：
          跳过项目名称/数量提取
  → 全部跑完：
      FLD .msg 解析（extract_msg提取HTML正文）
      → 从审批表格第8列提取审批价格
      → 差异 = 审批价格 - 订单净值
      → 写回Excel：E2E项目名 / 订单数量 / 项目总金额(审批) / 订单差异 / 支持文件名
```

## PM Tracking 匹配流程

```
透视表 E2E项目名
  → 在 NI PM Saving Tracking.xlsx 的 Base Data Sheet 中模糊匹配
  → 读取 Content 列，筛选以"装潢"开头的行（匹配装潢/装潢S/装潢B/装潢_外包板等）
       ├─ 1 行装潢 → ✅ 使用该行 Price/PlanCost
       ├─ 0 行装潢 → 回退到全部匹配行
       │    ├─ 1 行 → ✅ 使用
       │    ├─ ≥2 行 → ⚠ 跳过（弹窗报告：多行非装潢匹配）
       │    └─ 0 行 → ❌ 未匹配
       └─ ≥2 行装潢 → ⚠ 跳过（弹窗报告：多行装潢匹配）
  → 匹配成功时：
      总saving = 订单净值 × PlanCost ÷ Price
      SAVING差异 = 总saving - 订单净值
      Price=0或空 → 填入 "N/A"
```

## 关键机制

- **登录**：Microsoft SSO，auth.json持久化，过期自动提示重登
- **按钮定位**：标准点击 → JS点击 → dispatchEvent 三级兜底
- **防休眠**：搜索页复用不关闭
- **MSG解析**：extract_msg提取HTML正文 → 解析审批表格 → 取第8列审批价格
- **附件下载**：全部附件下载，按 PO 号建子文件夹，FLD/非FLD 分别存入 `downloads/FLD文件/` 和 `downloads/无FLD文件/`
- **项目名分文件夹**：非法字符替换为`_`
- **Content 筛选**：NI PM Saving Tracking 的 Content 列以"装潢"开头即匹配（装潢/装潢S/装潢B/装潢_外包板等）
- **多行跳过**：装潢匹配到多行 → 跳过不写 → 弹窗列出 PO号+项目名+原因
- **弹窗报告**：成功/失败/跳过计数 + 明细列表

## 注意事项

1. 运行前关闭Excel中的 EXPORT1 2 1.xlsx
2. 工厂清单：A=Plant, J=区域, K=分公司
3. 首次网站查询需手动完成 Microsoft 登录（最长等5分钟）
4. 网站查询现为全量模式（处理全部PO），不再截取前N个
5. PM Tracking 匹配依赖 Tracking 文件的 Content 列
6. 回滚：使用 `git reset --hard <tag>` 或 `git checkout <tag> -- <file>`
